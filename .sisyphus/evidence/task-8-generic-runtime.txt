Task 8 Evidence: Generic ApplicationRuntime
=============================================
Date: 2026-03-07

ApplicationRuntime is now fully generic:

```typescript
export class ApplicationRuntime<
  TConfig extends BaseRuntimeConfig = BaseRuntimeConfig,
  TContext extends BaseToolExecutionContext<TConfig> = BaseToolExecutionContext<TConfig>,
> {
  public readonly config: TConfig;
  public readonly registry: ToolRegistry<TContext>;
  private readonly logger: Logger;
  private readonly createContext: (toolName: string, requestId: string) => TContext;

  public constructor(options: RuntimeOptions<TConfig, TContext>) { ... }
}
```

Key design decisions:
1. contextFactory captured in closure to avoid TConfig appearing in contravariant position
   - Stored as `createContext: (toolName, requestId) => TContext` (no TConfig in signature)
   - This allows ApplicationRuntime<RuntimeConfig> to be assignable to ApplicationRuntime<BaseRuntimeConfig>
2. Default contextFactory creates base context: { requestId, toolName, config }
3. Custom contextFactory supports extended contexts (e.g., with traceId, auth info)

Tests verify:
- Default contextFactory creates proper context with requestId, toolName, config
- Custom contextFactory injects custom fields (customField, traceId)
- contextFactory called fresh for each executeTool invocation
- Generic types work with BaseRuntimeConfig and RuntimeConfig
